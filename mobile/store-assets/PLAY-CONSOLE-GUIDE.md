## Step 1: Google Play Console 접속

1. https://play.google.com/console 접속
2. Google 계정으로 로그인
3. 개발자 계정이 없다면 등록 ($25 일회성 비용)

---

## Step 2: 새 앱 만들기

1. **"앱 만들기"** 클릭
2. 앱 세부정보 입력:
   - **앱 이름**: 매일일독
   - **기본 언어**: 한국어 (ko-KR)
   - **앱 또는 게임**: 앱
   - **무료 또는 유료**: 무료
3. 개발자 프로그램 정책 동의 체크
4. **"앱 만들기"** 클릭

---

## Step 3: 스토어 등록정보 설정

### 3.1 기본 스토어 등록정보
**경로**: 스토어 등록정보 > 기본 스토어 등록정보

1. **앱 이름**: 매일일독
2. **짧은 설명** (80자):
   ```
   매일 성경을 읽고 진행 상황을 기록하는 성경통독 앱
   ```
3. **전체 설명**: `store-listing-ko.md`의 "전체 설명" 섹션 복사
4. **앱 아이콘**: `store-assets/icon-512.png` 업로드
5. **피처 그래픽**: (선택) 1024x500 이미지 업로드
6. **스크린샷**: `screenshots/` 폴더의 이미지 업로드
   - 휴대전화 스크린샷: 최소 2장, 최대 8장

### 3.2 스크린샷 업로드 순서
1. `01-home.png` - 홈 화면
2. `02-bible.png` - 성경 페이지
3. `03-plan.png` - 통독표
4. `04-hasena.png` - 하세나하시조
5. `05-login.png` - 로그인

---

## Step 4: 앱 콘텐츠 설정

### 4.1 개인정보처리방침
**경로**: 정책 > 앱 콘텐츠 > 개인정보처리방침

- **URL**: `https://maeil1dok.app/privacy`

### 4.2 광고
- **광고 포함 여부**: 아니요 (광고 없음)

### 4.3 앱 액세스 권한
- **제한된 액세스 없음** 또는 **로그인 필요**
- 로그인 필요 시 테스트 계정 제공 필요

### 4.4 콘텐츠 등급
1. 설문지 작성 시작
2. 카테고리: **참고/교육**
3. 질문에 정직하게 답변
   - 폭력적 콘텐츠: 없음
   - 성적 콘텐츠: 없음
   - 욕설: 없음
   - 약물: 없음
   - 도박: 없음
4. 등급 확인 (대부분 **전체 이용가** 예상)

### 4.5 타겟층
- **타겟 연령대**: 18세 이상 (또는 13세 이상)
- 어린이 대상 아님

### 4.6 뉴스 앱
- **뉴스 앱**: 아니요

### 4.7 COVID-19 관련
- 해당 없음

### 4.8 데이터 보안
1. **데이터 수집**: 예
2. **수집 데이터 유형**:
   - 이메일 주소 (계정 관리)
   - 이름/닉네임 (프로필)
3. **데이터 암호화**: 예 (HTTPS)
4. **데이터 삭제 요청**: 가능 (앱 내 계정 삭제)

---

## Step 5: 앱 릴리스 설정

### 5.1 프로덕션 트랙 선택
**경로**: 출시 > 프로덕션

1. **"새 버전 만들기"** 클릭
2. **App Bundle 업로드**: 빌드된 `.aab` 파일 업로드
3. **출시 노트** 작성:
   ```
   매일일독 첫 번째 릴리스
   - 매일 성경 읽기 기능
   - 통독 진행률 추적
   - 하세나하시조 영상 시청
   - 그룹 기능
   ```

### 5.2 출시 국가/지역
1. **"국가/지역 추가"** 클릭
2. **대한민국** 선택 (또는 전체 국가)

---

## Step 6: 검토 및 출시

### 6.1 대시보드 확인
모든 섹션이 녹색 체크 표시인지 확인:
- ✅ 앱 액세스
- ✅ 광고
- ✅ 콘텐츠 등급
- ✅ 타겟층
- ✅ 개인정보처리방침
- ✅ 데이터 보안
- ✅ 앱 카테고리
- ✅ 스토어 등록정보
- ✅ 스토어 설정

### 6.2 검토 제출
1. 모든 항목 완료 확인
2. **"검토를 위해 변경사항 제출"** 클릭
3. 검토 대기 (보통 1-3일, 최대 7일)
